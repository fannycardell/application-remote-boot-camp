Why code review?

In our fast track world it's easy to miss small details that can be of importance. A review means by its name "to look or view again". When coding this is super useful tool both for the code "maker" and the code "viewers". The "maker" will get feelback and suggestions on how to improve or change things and the "viewers" will learn things from the "maker" to use in their forthcoming coding. If the reviewing is made thoroughly and systematicalbly (even if it takes "extra time") it will gain the company by getting an overall increase in code quality in the end of the day.
